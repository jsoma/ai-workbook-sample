Building an AI demo, Part 3: Gradio components and page layout

We’ve done a pretty good job so far! We built an AI application from scratch in part one, then experimented with different Gradio outputs and models from Hugging Face in part two.

Now that we know how to put together a solid demo, it’s time to make it really shine! In this chapter we’re going to customize inputs and outputs even further, juggle multiple models all at once, and move away from a Gradio interface to a fully customized page layout.

Our starting point

We’re going to start from a text-to-text Gradio interface that translates text using a basic translation model.

import gradio as gr
from transformers import pipeline

pipe = pipeline("t5-small", model="t5-small")

def do_action(text):
    result = pipe(text)

    return result['translation_text']

iface = gr.Interface(fn=do_action, inputs="text", outputs="text")
iface.launch()

Nothing too zany: it’s an interface that takes in a text input, runs it through the translation pipeline, and then gives out the text translation. You’re asking for result['translation_text'] because result by itself just gives you some Python junk.

Using translations

Translation pipelines are kind of weird because you don’t just say pipeline("translation"), you have to explicitly state the from and to langauges.

# English to French
pipe = pipeline("translation_en_to_fr")
# Chinese to German
pipe = pipeline("translation_zh_to_de")

It takes some effort to uncover this, but it’s apparently on this one single page here

But let’s say instead of just translating to one language, we wanted to translate to many languages.

Multiple outputs are pretty common in AI demos! Besides side-by-side translation, you might take a piece of art and redraw it in a few artists’ styles or show the user several pieces of a multi-step process.

To adapt our code to output multiple translations, we need two things: some additional pipelines and a few more outputs.

import gradio as gr
from transformers import pipeline

pipe_fr = pipeline("translation_en_to_fr", model="t5-small")
pipe_de = pipeline("translation_en_to_de", model="t5-small")
pipe_zh = pipeline("translation_en_to_zh", model="Helsinki-NLP/opus-mt-en-zh")

def do_action(text):
    result_fr = pipe_fr(text)['translation_text']
    result_de = pipe_de(text)['translation_text']
    result_zh = pipe_zh(text)['translation_text']

    return result_fr, result_de, result_zh

iface = gr.Interface(fn=do_action, inputs="text", outputs=["text", "text", "text"])
iface.launch()

We could probably figure out the multiple pipelines part, but outputs piece might be surprising. All we had to do was pass a list to outputs=, and the ["text", "text", "text"] means “give us three text elements.

Same as before, we can use any of the shortcuts we wanted to display whatever is necessary. For example, we could use `[“text”, “label”, “image”]`` to display some text, a label score and an image.
